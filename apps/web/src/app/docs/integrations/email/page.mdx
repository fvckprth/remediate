## prerequisites

- an email provider that accepts file attachments — resend, sendgrid, postmark, ses, mailgun, or plain nodemailer all work
- a verified from address on that provider

`toEmail` returns a provider-agnostic payload — `subject`, `html`, `text`, `attachments` — so the code below works with any of them. the example uses resend.

## install

```bash
npm install remediate resend
```

## environment variables

```bash
# .env
RESEND_KEY=re_xxxxxxxxxxxx
FEEDBACK_FROM=feedback@yourapp.com
FEEDBACK_TO=team@yourapp.com
```

## client — mount the widget

```tsx
<Remediate endpoint="/api/feedback" />
```

## server — api route

`toEmail` returns ready-to-send content. most providers expect attachments as `Buffer`s, so convert each `Blob` with `arrayBuffer()`.

```ts
// app/api/feedback/route.ts
import { parseFeedback } from "remediate/server";
import { toEmail } from "remediate/email";
import { Resend } from "resend";

const resend = new Resend(process.env.RESEND_KEY!);

export async function POST(req: Request) {
  const feedback = await parseFeedback(req);
  const email = toEmail(feedback);

  await resend.emails.send({
    from: process.env.FEEDBACK_FROM!,
    to: process.env.FEEDBACK_TO!,
    subject: email.subject,
    html: email.html,
    text: email.text,
    attachments: await Promise.all(
      email.attachments.map(async (a) => ({
        filename: a.filename,
        content: Buffer.from(await a.content.arrayBuffer()),
      })),
    ),
  });

  return Response.json({ ok: true, id: feedback.submission.id });
}
```

## output shape

- `subject` — auto-derived title, or `Feedback from /path` if no text is present
- `html` — simple inline-styled html (no external css) so it renders cleanly in gmail, apple mail, and outlook
- `text` — plain text fallback mirroring the html
- `attachments` — screenshots, recordings, and voice notes as `{ filename, content, contentType }` tuples
