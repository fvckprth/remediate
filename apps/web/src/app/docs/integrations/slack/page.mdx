## prerequisites

- a slack workspace where you can install apps
- a slack bot token with `chat:write` and `files:write` scopes
- the channel id you want to post feedback into

create a slack app at [api.slack.com/apps](https://api.slack.com/apps), add the scopes above, install it to your workspace, and copy the bot user oauth token.

## install

```bash
npm install remediate @slack/web-api
```

## environment variables

```bash
# .env
SLACK_TOKEN=xoxb-xxxxxxxxxxxx
SLACK_CHANNEL_ID=C01234567
```

## client — mount the widget

point the widget at your api route. on submit, the widget POSTs everything as `FormData`.

```tsx
<Remediate endpoint="/api/feedback" />
```

## server — api route

parse the incoming submission, format it with `toSlackMessage`, and use the official slack web client to post the message and upload any attached files.

```ts
// app/api/feedback/route.ts
import { parseFeedback } from "remediate/server";
import { toSlackMessage } from "remediate/slack";
import { WebClient } from "@slack/web-api";

const slack = new WebClient(process.env.SLACK_TOKEN);

export async function POST(req: Request) {
  const feedback = await parseFeedback(req);
  const msg = toSlackMessage(feedback);

  await slack.chat.postMessage({
    channel: process.env.SLACK_CHANNEL_ID!,
    blocks: msg.blocks,
    text: msg.text,
  });

  for (const file of msg.files) {
    await slack.filesUploadV2({
      channel_id: process.env.SLACK_CHANNEL_ID!,
      file: file.content,
      filename: file.filename,
      title: file.title,
    });
  }

  return Response.json({ ok: true, id: feedback.submission.id });
}
```

## output shape

`toSlackMessage` returns a `SlackMessagePayload` with three parts:

- `blocks` — slack block kit blocks: a header with the feedback title, a section block with the page url, one block per feedback item (with priority tags), and a context footer with browser / os / viewport
- `text` — a plain text fallback for notifications
- `files` — screenshots, recordings, and voice notes as `Blob`s ready to hand to `filesUploadV2`

item titles are auto-derived from the first text note, annotation note, or additional text in the submission. priorities (none / low / medium / high / urgent) are rendered inline as `[priority]` tags.
