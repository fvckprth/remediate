the root object is a `FeedbackSubmission`. here's the shape at a glance:

```
FeedbackSubmission
├── id, url, timestamp
├── environment { browser, os, viewport, screen, dpr, language, timezone, colorScheme }
├── items[]
│   ├── photo — screenshot blob + selection area
│   ├── video — recording blob + duration
│   ├── annotation — element fingerprint + note
│   ├── textNote — plain text
│   └── voiceNote — audio blob + duration
└── metadata — your custom data
```

## FeedbackSubmission

```ts
interface FeedbackSubmission {
  id: string;
  url: string;
  timestamp: string;
  environment: {
    userAgent: string;
    browser: { name: string; version: string };
    os: { name: string; version: string };
    viewport: { width: number; height: number };
    screen: { width: number; height: number };
    devicePixelRatio: number;
    language: string;
    timezone: string;
    colorScheme: "light" | "dark";
  };
  items: FeedbackItem[];
  metadata: Record<string, unknown>;
}
```

- `id` — uuid generated at submission time
- `url` — full page url where the widget was used
- `timestamp` — iso-8601 string
- `environment` — browser, os, viewport, screen, dpr, language, timezone, and color scheme, sniffed at submission time
- `items` — a mixed array of `FeedbackItem` variants, in the order the user added them
- `metadata` — whatever you passed via the widget's `metadata` prop, verbatim

## FeedbackItem

every item is a tagged union on `type` and shares a common base.

```ts
type FeedbackItem =
  | PhotoCapture
  | VideoCapture
  | AnnotationItem
  | TextNoteItem
  | VoiceNoteItem;
```

```ts
interface FeedbackItemBase {
  id: string;
  index: number;              // 1-based order in the submission
  timestamp: number;          // Date.now()
  additionalText: string;     // optional caption
  priority: "none" | "low" | "medium" | "high" | "urgent";
}
```

### photo

a screenshot of a selected region of the page, rendered client-side from the dom.

```ts
interface PhotoCapture extends FeedbackItemBase {
  type: "photo";
  area: SelectionArea;
  blob?: Blob;                // PNG
}
```

### video

a screen recording via `getDisplayMedia`. the blob is a webm file. requires https.

```ts
interface VideoCapture extends FeedbackItemBase {
  type: "video";
  duration: number;           // seconds
  area: SelectionArea;
  blob?: Blob;                // WebM
}
```

### annotation

a pin dropped on a specific dom element. `element` is a full fingerprint: selector, xpath-style path, computed styles, bounding rect, nearby text, and every attribute on the element.

```ts
interface AnnotationItem extends FeedbackItemBase {
  type: "annotation";
  element: ElementCapture;
  note: string;
  clickOffset: { x: number; y: number };
}
```

### textNote

a plain text note from the user.

```ts
interface TextNoteItem extends FeedbackItemBase {
  type: "textNote";
  text: string;
}
```

### voiceNote

a microphone recording via `getUserMedia`. the blob is a webm audio file. requires https.

```ts
interface VoiceNoteItem extends FeedbackItemBase {
  type: "voiceNote";
  duration: number;           // seconds
  blob?: Blob;                // WebM audio
}
```

## ElementCapture

embedded inside every annotation item. note that `nearbyText` captures visible text around the annotated element — if your page displays sensitive data, it may appear in the payload.

```ts
interface ElementCapture {
  selector: string;
  name: string;
  elementPath: string;
  boundingRect: { x: number; y: number; width: number; height: number };
  nearbyText: string;
  cssClasses: string;
  attributes: Record<string, string>;
  computedStyles: Record<string, string>;
}
```

## SelectionArea

embedded inside photo and video items. coordinates are in css pixels, relative to the page viewport at capture time.

```ts
interface SelectionArea {
  x: number;
  y: number;
  width: number;
  height: number;
}
```

## blobs on the wire

the `blob` property is only populated in the `onSubmit` path. when using `endpoint` mode, blobs are stripped from the json and uploaded as separate `FormData` parts. on the server, call `parseFeedback(req)` from `remediate/server` to get a `ParsedFeedback` with a `submission` and a `files` map.
